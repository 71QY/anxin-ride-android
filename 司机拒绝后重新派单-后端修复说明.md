# 司机拒绝后重新派单功能 - 后端修复说明

## 需求描述

当用户（无论是长辈端还是普通用户端）拒绝司机接单后，系统应该在**10秒后自动推送新的司机申请**，形成循环直到用户取消订单。

## 当前问题

- ❌ 用户拒绝司机后，没有后续的新司机推送
- ❌ 订单停留在"待确认"状态，但没有新的司机来申请
- ❌ 用户只能手动取消订单，无法继续等待其他司机

## 期望行为

### 正常流程

```
1. 用户创建订单 → 订单状态: 待确认(status=0)
2. 后端推送司机A申请 → 前端显示确认弹窗
3. 用户拒绝司机A → 发送拒绝请求到后端
4. 后端记录拒绝原因，订单保持status=0
5. ⭐ 后端在10秒后推送司机B申请 → 前端再次显示确认弹窗
6. 用户拒绝司机B → 重复步骤3-5
7. 直到用户同意某个司机 OR 用户主动取消订单
```

### 边界情况

1. **用户取消订单**：停止所有后续的司机推送
2. **无可用司机**：如果10秒内没有新司机，可以延长等待时间或提示用户
3. **订单超时**：可以设置最大等待时间（如30分钟），超时后自动取消

### 2. 实现定时推送新司机的任务

**方案A: 使用延迟队列（推荐）**

```java
@Component
public class DriverPushScheduler {
    
    @Autowired
    private WebSocketService webSocketService;
    
    @Autowired
    private OrderRepository orderRepository;
    
    @Autowired
    private DriverService driverService;
    
    /**
     * 调度下一次司机推送
     * @param orderId 订单ID
     * @param delaySeconds 延迟秒数
     */
    public void scheduleNextDriverPush(Long orderId, int delaySeconds) {
        // 使用延迟队列或定时任务
        CompletableFuture.delayedExecutor(delaySeconds, TimeUnit.SECONDS)
            .execute(() -> pushNextDriver(orderId));
    }
    
    /**
     * 推送下一个司机
     */
    private void pushNextDriver(Long orderId) {
        try {
            // 1. 检查订单是否仍然有效（未被取消）
            Order order = orderRepository.findById(orderId).orElse(null);
            if (order == null || order.getStatus() != 0) {
                log.info("订单 {} 已取消或状态变更，停止推送", orderId);
                return;
            }
            
            // 2. 查找下一个可用司机（排除已拒绝的司机）
            List<Long> rejectedDriverIds = getRejectedDriverIds(orderId);
            Driver nextDriver = driverService.findNextAvailableDriver(
                order.getStartLat(), 
                order.getStartLng(),
                rejectedDriverIds  // ⭐ 排除已拒绝的司机
            );
            
            if (nextDriver == null) {
                log.warn("订单 {} 暂无可用司机，10秒后重试", orderId);
                // ⭐ 如果没有司机，10秒后重试
                scheduleNextDriverPush(orderId, 10);
                return;
            }
            
            // 3. 构造推送消息
            WsMessage message = WsMessage.builder()
                .type("DRIVER_REQUEST")
                .orderId(orderId)
                .driverId(nextDriver.getId())
                .driverName(nextDriver.getName())
                .driverPhone(nextDriver.getPhone())
                .driverAvatar(nextDriver.getAvatar())
                .carNo(nextDriver.getCarNo())
                .carType(nextDriver.getCarType())
                .carColor(nextDriver.getCarColor())
                .rating(nextDriver.getRating())
                .message("有新的司机接单，是否同意？")
                .build();
            
            // 4. 推送给用户
            String userSessionId = "user_" + order.getUserId();
            webSocketService.sendMessage(userSessionId, message);
            
            log.info("✅ 已向用户 userId={} 推送司机 {} 的申请", 
                order.getUserId(), nextDriver.getName());
                
        } catch (Exception e) {
            log.error("❌ 推送司机失败，订单ID: {}", orderId, e);
            // ⭐ 失败后10秒重试
            scheduleNextDriverPush(orderId, 10);
        }
    }
    
    /**
     * 获取订单已拒绝的司机ID列表
     */
    private List<Long> getRejectedDriverIds(Long orderId) {
        // 从订单的拒绝记录中查询
        return orderRejectRecordRepository.findByOrderId(orderId)
            .stream()
            .map(record -> record.getDriverId())
            .collect(Collectors.toList());
    }
}
```

**方案B: 使用Spring @Scheduled（备选）**

```java
@Component
public class DriverPushTask {
    
    @Autowired
    private OrderRepository orderRepository;
    
    /**
     * 每分钟扫描一次需要重新推送的订单
     */
    @Scheduled(fixedRate = 60000)  // 每60秒执行
    public void scanPendingOrders() {
        LocalDateTime tenSecondsAgo = LocalDateTime.now().minusSeconds(10);
        
        // 查询10秒前被拒绝且仍在等待的订单
        List<Order> pendingOrders = orderRepository.findOrdersNeedRetry(tenSecondsAgo);
        
        for (Order order : pendingOrders) {
            pushNextDriver(order.getId());
        }
    }
}
```

### 4. 订单状态管理

**订单状态定义**:
```java
public enum OrderStatus {
    PENDING_CONFIRM(0, "待确认"),      // ⭐ 用户拒绝司机后回到此状态
    CONFIRMED(1, "已确认"),
    WAITING_DRIVER(2, "等待司机"),
    DRIVER_ACCEPTED(3, "司机已接单"),
    IN_PROGRESS(4, "行程中"),
    COMPLETED(5, "已完成"),
    CANCELLED(6, "已取消"),
    REJECTED(7, "已拒绝");  // ⭐ 这个状态可能不再使用，或者仅用于最终拒绝
    
    private final int code;
    private final String desc;
    
    OrderStatus(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }
}
```

**关键点**:
- 用户拒绝司机后，订单状态**保持为 0 (待确认)**，不要改为 7 (已拒绝)
- 只有用户主动取消订单时，才将状态改为 6 (已取消)
- 可以考虑废弃状态 7 (已拒绝)，或者将其定义为"用户最终拒绝叫车"

### 5. 取消订单时清理定时任务

**接口**: `POST /api/order/cancel/{orderId}`

```java
@PostMapping("/cancel/{orderId}")
public Result<?> cancelOrder(@PathVariable Long orderId) {
    try {
        Order order = orderRepository.findById(orderId).orElse(null);
        if (order == null) {
            return Result.error("订单不存在");
        }
        
        // 1. 更新订单状态
        order.setStatus(6);  // 已取消
        order.setCancelTime(LocalDateTime.now());
        orderRepository.save(order);
        
        // 2. ⭐ 关键：取消所有待执行的司机推送任务
        driverPushScheduler.cancelScheduledTasks(orderId);
        
        // 3. 通知前端
        WsMessage message = WsMessage.builder()
            .type("ORDER_CANCELLED")
            .orderId(orderId)
            .message("订单已取消")
            .build();
        
        String userSessionId = "user_" + order.getUserId();
        webSocketService.sendMessage(userSessionId, message);
        
        return Result.success("订单已取消");
        
    } catch (Exception e) {
        log.error("❌ 取消订单失败", e);
        return Result.error("取消订单失败");
    }
}
```

## 性能优化建议

### 1. 防止频繁推送

```java
// 限制每个订单的最大重试次数
private static final int MAX_RETRY_COUNT = 30;  // 最多重试30次（5分钟）

public void scheduleNextDriverPush(Long orderId, int delaySeconds, int retryCount) {
    if (retryCount >= MAX_RETRY_COUNT) {
        log.warn("订单 {} 已达到最大重试次数，停止推送", orderId);
        // 可以发送通知告知用户暂无可用司机
        return;
    }
    
    CompletableFuture.delayedExecutor(delaySeconds, TimeUnit.SECONDS)
        .execute(() -> pushNextDriver(orderId, retryCount + 1));
}
```

### 2. 智能退避策略

```java
// 第一次拒绝：10秒后推送
// 第二次拒绝：15秒后推送
// 第三次拒绝：20秒后推送
// ... 逐步增加间隔，避免频繁打扰用户
int nextDelay = Math.min(10 + (retryCount * 5), 60);  // 最大60秒
scheduleNextDriverPush(orderId, nextDelay, retryCount + 1);
```

### 3. 并发控制

```java
// 使用 ConcurrentHashMap 存储每个订单的定时任务
private final ConcurrentHashMap<Long, ScheduledFuture<?>> scheduledTasks = new ConcurrentHashMap<>();

public void scheduleNextDriverPush(Long orderId, int delaySeconds) {
    // 取消之前的任务（如果有）
    cancelScheduledTask(orderId);
    
    ScheduledFuture<?> future = scheduler.schedule(
        () -> pushNextDriver(orderId),
        delaySeconds,
        TimeUnit.SECONDS
    );
    
    scheduledTasks.put(orderId, future);
}

public void cancelScheduledTasks(Long orderId) {
    ScheduledFuture<?> future = scheduledTasks.remove(orderId);
    if (future != null) {
        future.cancel(false);
    }
}
```

## 总结

### 核心改动点

1. ✅ **拒绝司机后订单状态保持为0**（待确认），不要改为7（已拒绝）
2. ✅ **记录拒绝历史**，避免重复推送同一司机
3. ✅ **实现定时推送机制**，10秒后推送新司机
4. ✅ **取消订单时清理定时任务**，避免无效推送
5. ✅ **添加重试次数限制**，防止无限循环

### 预期效果

- ✅ 用户拒绝司机后，10秒内会收到新司机申请
- ✅ 可以连续拒绝多次，每次都会推送新司机
- ✅ 用户取消订单后，立即停止推送
- ✅ 适用于长辈端和普通用户端

### 注意事项

⚠️ **重要**: 这是一个**后端主导**的功能，前端只需要接收WebSocket消息即可，无需额外开发。

⚠️ **兼容性**: 确保旧的订单数据也能正常工作，可能需要数据迁移脚本。

⚠️ **监控**: 建议添加日志和监控，统计平均等待时间、重试次数等指标。
