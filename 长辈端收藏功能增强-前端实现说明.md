# 长辈端收藏功能增强 - 前端实现说明

## 📋 功能概述

本次更新为长辈端收藏功能实现了以下5个核心能力：

1. ✅ **快速指定目的地**：长辈点击"发送目的地"按钮，一键发送给亲友代叫车
2. ✅ **查看详细信息**：长辈点击"查看详情"按钮，查看地址、电话、简介等信息
3. ✅ **行程到达确认**：长辈点击"✅ 已到达"按钮，立即通知所有绑定的亲友
4. ✅ **语音播报**：长辈点击收藏卡片时，App自动语音播报地点信息
5. ✅ **行程凭证记录**：系统自动记录每次访问收藏地点的时间和位置

```kotlin
fun initTTS(context: Context) {
    textToSpeech = TextToSpeech(context) { status ->
        if (status == TextToSpeech.SUCCESS) {
            val result = textToSpeech?.setLanguage(Locale.CHINESE)
            ttsInitialized = (result != TextToSpeech.LANG_MISSING_DATA && 
                             result != TextToSpeech.LANG_NOT_SUPPORTED)
        }
    }
}
```

2. **语音播报地点信息**
```kotlin
fun speakLocationInfo(name: String, address: String, phone: String? = null) {
    val message = buildString {
        append("这是$name")
        append("，地址：$address")
        phone?.let {
            if (it.isNotEmpty()) append("，电话：$it")
        }
    }
    textToSpeech?.speak(message, TextToSpeech.QUEUE_FLUSH, null, null)
}
```

3. **分享收藏地点给亲友**
```kotlin
fun shareFavoriteToGuardian(
    favoriteId: Long,
    guardianUserId: Long,
    onSuccess: () -> Unit = {},
    onError: (String) -> Unit = {}
) {
    viewModelScope.launch {
        try {
            val result = apiService.shareFavoriteToGuardian(favoriteId, guardianUserId)
            if (result.isSuccess()) onSuccess() else onError(result.message ?: "分享失败")
        } catch (e: Exception) {
            onError(e.message ?: "网络异常")
        }
    }
}
```

4. **确认到达目的地**

fun confirmArrival(

## 📡 五、WebSocket消息处理

### 5.1 发送给亲友的消息格式

当前端调用 `shareFavoriteToGuardian` API后，后端会通过WebSocket向亲友推送：

```json
{
  "type": "FAVORITE_DESTINATION_SHARED",
  "elderId": 654321,
  "elderName": "张三",
  "favoriteId": 789,
  "destinationName": "市人民医院",
  "destinationAddress": "北京市朝阳区XX路123号",
  "latitude": 39.9042,
  "longitude": 116.4074,
  "phone": "010-12345678",
  "description": "三甲医院，心血管科专家门诊",
  "timestamp": 1713412200000
}
```

### 5.2 到达确认消息格式

当前端调用 `confirmArrival` API后，后端会通过WebSocket向所有亲友推送：

```json
{
  "type": "ARRIVAL_CONFIRMED",
  "elderId": 654321,
  "elderName": "张三",
  "favoriteId": 789,
  "destinationName": "市人民医院",
  "orderId": 999,
  "arrivedAt": "2026-04-18T11:00:00",
  "latitude": 39.9042,
  "longitude": 116.4074,
  "timestamp": 1713415200000
}
```

### 5.3 亲友端接收处理

亲友端的HomeViewModel需要监听这两种消息类型：

```kotlin
// 在HomeViewModel中处理WebSocket消息
when (message.type) {
    "FAVORITE_DESTINATION_SHARED" -> {
        // 显示代叫车确认对话框
        showProxyOrderDialog(message)
    }
    "ARRIVAL_CONFIRMED" -> {
        // 显示到达通知
        showArrivalNotification(message)
    }
}
```

## 📱 七、用户体验优化建议

### 7.1 语音播报优化

- **语速调整**：可根据用户反馈调整TTS语速
- **音量控制**：确保播报音量足够大，适合老年人
- **打断机制**：如果正在播报时再次点击，应该打断前一次播报，立即播放新的

### 7.2 按钮反馈

- **触觉反馈**：点击大按钮时，可添加震动反馈
- **视觉反馈**：按钮按下时有明显的颜色变化
- **加载状态**：分享或确认到达时，显示加载动画

### 7.3 错误提示

- **友好文案**：避免技术术语，使用老年人能理解的语言
- **语音提示**：重要操作失败时，可考虑语音提示
- **重试机制**：网络失败时，提供"重试"按钮
