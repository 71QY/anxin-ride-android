# 代码修改后不生效问题解决方案

## 问题描述
修改代码后重新运行应用，看到的还是旧代码。只有卸载应用后重新安装才能看到新代码。

## 原因分析
1. **Gradle 构建缓存** - Gradle 使用了旧的编译产物
2. **Android Studio 增量更新失败** - Apply Changes 功能有时不可靠
3. **设备上的 APK 未完全替换** - 直接运行可能没有正确覆盖旧版本

## 解决方案

### 方案一：使用快速重建脚本（推荐日常开发）

```powershell
.\quick_rebuild.ps1
```

这个脚本会：
- 停止 Gradle Daemon
- 清理 app/build 目录
- 清理 Kotlin 缓存
- 重新构建并安装应用

**优点**：速度快，适合频繁使用  
**缺点**：不会卸载应用，某些情况下可能仍有残留

---

### 方案二：使用强制重装脚本（最彻底）

```powershell
.\force_reinstall.ps1
```

这个脚本会：
- 停止 Gradle Daemon
- **卸载设备上的旧应用**
- 清理所有构建目录和缓存
- 重新构建并安装应用

**优点**：最彻底，保证看到最新代码  
**缺点**：速度较慢，会清除应用数据

---

### 方案三：在 Android Studio 中手动操作

1. **Build → Clean Project**
2. **Build → Rebuild Project**
3. **Run → Run 'app'** （或按 Shift+F10）

如果还不行：
4. **File → Invalidate Caches / Restart → Invalidate and Restart**

---

### 方案四：命令行方式

```powershell
# 清理并重新构建
.\gradlew clean assembleDebug

# 安装到设备
.\gradlew installDebug
```

或者一条命令：
```powershell
.\gradlew clean installDebug
```

## 配置优化说明

已修改 `gradle.properties` 配置：

```properties
# ✅ 启用 Gradle 构建缓存（提高编译速度）
org.gradle.caching=true

# ✅ 启用 Kotlin 增量编译（可靠且快速）
kotlin.incremental=true

# ✅ 启用 Kotlin 缓存
kotlin.caching.enabled=true
```

**之前的问题**：禁用了所有缓存，导致每次都是全量编译，反而更容易出错。  
**现在的配置**：启用缓存，利用 Gradle 和 Kotlin 的增量编译机制，既快又可靠。

## 日常开发建议

### 正常情况（90% 的场景）
直接点击 Android Studio 的 **Run** 按钮（▶️）或按 **Shift+F10**

增量编译会自动检测变化并只编译修改的文件，速度很快。

### 遇到代码不更新时
1. 先尝试：**Build → Rebuild Project**
2. 如果不行：运行 `.\quick_rebuild.ps1`
3. 如果还不行：运行 `.\force_reinstall.ps1`

### 提交代码前
```powershell
.\gradlew clean build
```

确保没有编译错误。

---

## 为什么卸载重装能看到新代码？

因为卸载会：
1. 删除应用的所有数据和缓存
2. 强制 Android 系统接受全新的 APK
3. 清除所有可能的运行时缓存

我们的脚本模拟了这个过程，但不需要你手动卸载。

---

## 常见问题

### Q: 为什么不能每次都自动清理？
A: 清理后需要全量编译，速度慢（可能需要几分钟）。增量编译只需几秒。

### Q: 哪个脚本更好？
A: 
- 日常开发：直接用 Run 按钮
- 偶尔不更新：`quick_rebuild.ps1`
- 彻底解决：`force_reinstall.ps1`

### Q: 能否禁用 Instant Run/Apply Changes？
A: 已经在 `gradle.properties` 中禁用了：
```properties
android.enableInstantRun=false
```

## 总结

| 场景 | 推荐操作 | 耗时 |
|------|---------|------|
| 日常开发 | Run 按钮 (Shift+F10) | ⚡ 5-15秒 |
| 代码不更新 | `quick_rebuild.ps1` | 🕐 30-60秒 |
| 顽固问题 | `force_reinstall.ps1` | 🕑 2-5分钟 |
| 提交前检查 | `gradlew clean build` | 🕑 2-5分钟 |

**核心原则**：优先使用增量编译，必要时才清理缓存。
