# 批量图片发送 - 后端配合说明

## 📋 概述

前端已实现多图片批量发送功能，用户一次可选择最多3张图片，合并为一条WebSocket消息发送给后端。需要后端配合解析新增的字段并返回识别结果。

## 📝 后端需要处理的字段

### 新增字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `additionalImages` | Array\<String\> | 否 | 额外的图片Base64数组，最多2个元素 |
| `imageCount` | Integer | 是 | 图片总数（1-3） |

### 处理逻辑

```java
// 伪代码示例
public void handleImageMessage(JSONObject message) {
    String mainImage = message.getString("imageBase64");  // 主图

    List<String> allImages = new ArrayList<>();
    allImages.add(mainImage);  // 添加主图
```

## ⚠️ 重要注意事项

### 1. 图片数量限制

- **最小值**：1张
- **最大值**：3张
- 后端应验证 `imageCount` 字段与实际接收的图片数量是否一致

### 2. 图片大小限制

- 单张图片最大：**10MB**（前端已压缩到1.5MB左右）
- 总大小限制：建议不超过 **15MB**（3张 × 5MB）

### 3. 识别策略

**推荐方案A（分别识别）**：
- 对每张图片单独调用OCR/图像识别API
- 将所有识别结果合并为一个综合回复
- 优点：准确度高，可以针对每张图片给出详细分析

**推荐方案B（组合识别）**：
- 将多张图片拼接或作为一组输入给AI模型
- 让AI理解图片之间的关系
- 优点：可以理解图片之间的上下文关系

### 4. 响应时间

- 单张图片识别：建议 **3-5秒** 内返回
- 多张图片识别：建议 **5-10秒** 内返回
- 如果识别时间较长，可以先返回一个"正在识别中..."的提示消息
