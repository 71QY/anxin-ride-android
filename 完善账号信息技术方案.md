# 后端紧急修复 - 亲情守护功能

## 🚨 紧急问题

**问题描述**：`register-elder` 接口调用失败，报错：
```
Unknown column 'elder_id_card' in 'field list'
```

**原因**：`family_guard` 表缺少 `elder_id_card` 字段

**影响**：
- 无法帮长辈注册账号
- 用户用手机号登录后创建了普通账号（guard_mode=0）
- 长辈无法进入长辈模式

---

## 🔧 修复步骤（按顺序执行）

### 第1步：添加缺失的数据库字段 ⭐ 最关键

```sql
USE anxin_travel;

-- 添加 elder_id_card 字段到 family_guard 表
ALTER TABLE `family_guard` 
ADD COLUMN `elder_id_card` VARCHAR(18) DEFAULT NULL COMMENT '长辈身份证号' 
AFTER `elder_name`;

-- 验证添加成功
SHOW COLUMNS FROM `family_guard` LIKE 'elder_id_card';
```

**预期输出**：
```
+---------------+-------------+------+-----+---------+-------+
| Field         | Type        | Null | Key | Default | Extra |
+---------------+-------------+------+-----+---------+-------+

+---------------+-------------+------+-----+---------+-------+
```

# 新用户注册完整性校验 - 技术方案（待实施）

## 📋 需求说明

新注册的普通用户账号必须设置**密码**和**昵称**才算注册完成，否则不允许进入应用主界面。

---

## 🔧 后端实现方案

### 1. 数据库修改

在 `user` 表中添加字段：

```sql
ALTER TABLE user ADD COLUMN is_completed TINYINT(1) DEFAULT 0 COMMENT '账号是否完善：0-未完善 1-已完善';
ALTER TABLE user ADD COLUMN nickname VARCHAR(50) NULL COMMENT '用户昵称';
```

**说明**：
- `is_completed = 0`：账号未完善，需要引导用户设置密码和昵称
- `is_completed = 1`：账号已完善，可以正常使用
- 注册时默认为 `0`
- 完善后更新为 `1`

---

### 2. 接口修改

#### 2.1 注册接口 `POST /api/auth/register`

**当前行为**：
- 注册成功后直接返回 token，允许登录

**修改后行为**：
- 注册成功后设置 `is_completed = 0`
- 返回 token 但标记账号未完善

**响应示例**：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userId": 12345,
    "phone": "13800138000",
    "isGuarded": 0,
    "guardMode": 0,
    "isCompleted": 0  // ⭐ 新增：标记账号未完善
  }
}
```

---

#### 2.2 登录接口 `POST /api/auth/login`

**当前行为**：
- 登录成功后直接返回 token

**修改后行为**：
- 检查用户的 `is_completed` 字段
- 如果为 `0`，返回特殊状态码或标记

**响应示例（账号未完善）**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userId": 12345,
    "phone": "13800138000",
    "isGuarded": 0,
    "guardMode": 0,
    "isCompleted": 0  // ⭐ 新增：标记账号未完善
  }
}
```

**或者返回特殊错误码**（推荐）：
```json
{
  "code": 403,
  "message": "账号未完善，请先设置密码和昵称",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userId": 12345,
    "phone": "13800138000",
    "isGuarded": 0,
    "guardMode": 0,
    "isCompleted": 0
  }
}
```

---

#### 2.3 新增：完善账号接口 `POST /api/user/complete-profile`

**请求**：
```
POST /api/user/complete-profile
Content-Type: application/json
Authorization: Bearer {token}
或
X-User-Id: {userId}

{
  "password": "Abc@123456",      // 必填：密码（10位，包含字母和特殊符号）
  "nickname": "张三"              // 必填：昵称（1-20字符）
}
```

**响应（成功）**：
```json
{
  "code": 200,
  "message": "账号完善成功",
  "data": null
}
```

**响应（失败）**：
```json
{
  "code": 400,
  "message": "密码格式不正确，必须是10位且包含字母和特殊符号",
  "data": null
}
```

**业务逻辑**：
1. 验证密码格式（10位，包含字母和特殊符号）
2. 验证昵称不为空（1-20字符）
3. 更新 user 表：
   ```sql
   UPDATE user 
   SET password = MD5(?), 
       nickname = ?, 
       is_completed = 1 
   WHERE id = ?
   ```
4. 返回成功

---

### 3. 密码加密规则

**重要**：确保注册和完善账号时使用相同的密码加密方式。

建议使用：
- BCrypt（推荐）
- 或 MD5 + Salt

---

### 4. 权限控制

以下接口需要检查 `is_completed`：

| 接口 | 是否检查 | 说明 |
|------|---------|------|
| `/api/user/profile` | ❌ 不检查 | 允许查看基本信息 |
| `/api/user/avatar` | ❌ 不检查 | 允许上传头像 |
| `/api/order/*` | ✅ 检查 | 下单前必须完善 |
| `/api/map/*` | ✅ 检查 | 使用地图前必须完善 |
| `/api/agent/*` | ✅ 检查 | 使用智能体前必须完善 |

**检查逻辑**：
```java
if (user.getIsCompleted() == 0) {
    throw new BusinessException(403, "账号未完善，请先设置密码和昵称");
}
```

## 🔄 完整流程

### 场景1：新用户注册

## ⚠️ 注意事项

1. **向后兼容**：
   - 对于已存在的老用户，执行 SQL 批量更新：
