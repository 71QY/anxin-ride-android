## 🎯 用户操作流程

### 场景 1: 添加收藏
1. 用户点击底部导航"收藏"标签
2. 如果为空，显示"暂无收藏"引导界面
3. 点击"添加收藏"按钮或右上角"+"号
4. 弹出对话框，输入：
   - 地点名称（必填）
   - 详细地址
   - 地点类型（家/公司/医院/其他）
5. 点击"保存"，调用 `POST /api/favorites`
6. 成功后自动刷新列表，Toast 提示"添加成功"

### 场景 2: 编辑收藏
1. 在收藏列表中点击某个收藏项右侧的"编辑"图标
2. 弹出对话框，显示当前信息
3. 修改内容后点击"保存"
4. 调用 `PUT /api/favorites`
5. 成功后刷新列表，Toast 提示"更新成功"

### 场景 3: 删除收藏
1. 点击收藏项右侧的"删除"图标（红色）
2. 弹出二次确认对话框："确定要删除'XXX'吗？"
3. 点击"删除"按钮
4. 调用 `DELETE /api/favorites/{id}`
5. 成功后刷新列表，Toast 提示"删除成功"

### 场景 4: 一键叫车
1. 点击收藏列表中的任意一项
2. 自动跳转到"首页"标签
3. HomeViewModel 自动填充：
   - 目的地名称
   - 经纬度坐标
4. 用户可直接点击"一键叫车"按钮发起订单

## 📡 后端 API 对接说明

### 基础配置
- **Base URL**: `http://192.168.189.80:8080/api/`
- **鉴权方式**: Header 中携带 `Authorization: Bearer <token>`
- **Token 管理**: 由 AuthInterceptor 自动附加，前端无需手动处理

### 接口 1: 获取收藏列表
```
GET /api/favorites
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "name": "家",
      "address": "广东省潮州市湘桥区XX路XX号",
      "latitude": 23.656,
      "longitude": 116.622,
      "type": "HOME",
      "updatedAt": "2026-04-17T10:00:00"
    },
    {
      "id": 2,
      "name": "市人民医院",
      "address": "广东省潮州市XX区XX大道",
      "latitude": 23.660,
      "longitude": 116.630,
      "type": "HOSPITAL",
      "updatedAt": "2026-04-17T11:30:00"
    }
  ]
}
```

**注意事项**:
- 后端需根据 Token 中的 userId 自动查询该用户的收藏
- 按 `updatedAt` 倒序排列（最新的在前）
- 如果用户无收藏，返回空数组 `[]`

### 接口 2: 添加收藏
```
POST /api/favorites
Content-Type: application/json
```

**请求体**:
```json
{
  "name": "市人民医院",
  "address": "广东省潮州市XX区XX大道",
  "latitude": 23.660,
  "longitude": 116.630,
  "type": "HOSPITAL"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 3,
    "name": "市人民医院",
    "address": "广东省潮州市XX区XX大道",
    "latitude": 23.660,
    "longitude": 116.630,
    "type": "HOSPITAL",
    "updatedAt": "2026-04-17T12:00:00"
  }

```

**注意事项**:
- `userId` 由后端从 Token 中自动获取，**前端不传递**
- `type` 可选值：`HOME`, `COMPANY`, `HOSPITAL`, `CUSTOM`
- 后端需校验坐标范围（纬度 -90~90，经度 -180~180）
- 建议限制每个用户最多收藏 50 个地点

### 接口 4: 删除收藏
```
DELETE /api/favorites/{id}
```

**路径参数**:
- `id`: 收藏记录的 ID（Long 类型）

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

**注意事项**:
- 后端需校验 `userId` 与收藏记录的归属关系
- 删除成功后返回 200，data 可为 null
- 如果 ID 不存在，返回 404 错误

## ⚠️ 前端特殊要求

### 1. 数据库表结构建议
```sql
CREATE TABLE user_favorites (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    name VARCHAR(100) NOT NULL COMMENT '地点名称',
    address VARCHAR(500) NOT NULL COMMENT '详细地址',
    latitude DOUBLE NOT NULL COMMENT '纬度',
    longitude DOUBLE NOT NULL COMMENT '经度',
    type VARCHAR(20) DEFAULT 'CUSTOM' COMMENT '类型: HOME/COMPANY/HOSPITAL/CUSTOM',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    INDEX idx_user_id (user_id),
    INDEX idx_type (type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户收藏地点表';
```

### 2. 错误码规范
前端期望的错误响应格式：
```json
{
  "code": 400,
  "message": "具体的错误描述信息",
  "data": null
}
```

常见错误码：
- `200`: 成功
- `400`: 请求参数错误（如坐标超出范围）
- `401`: 未登录或 Token 失效
- `403`: 无权操作（尝试修改他人收藏）
- `404`: 收藏记录不存在
- `500`: 服务器内部错误

### 3. 数据校验规则
后端建议实现的校验：
- `name`: 非空，最大长度 100 字符
- `address`: 非空，最大长度 500 字符
- `latitude`: -90.0 ~ 90.0
- `longitude`: -180.0 ~ 180.0
- `type`: 必须为 `HOME`/`COMPANY`/`HOSPITAL`/`CUSTOM` 之一
