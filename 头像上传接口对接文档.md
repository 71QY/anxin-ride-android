# 头像上传接口对接文档（Android 前端）

## 📋 问题描述

**现象**：Android 端调用头像上传接口后，没有任何日志输出，疑似请求未到达后端或响应异常。

**影响**：用户无法上传/修改头像。

---

## 🔧 前端实现详情

### 1. 接口定义

**文件位置**：`app/src/main/java/com/example/myapplication/core/network/ApiService.kt`

```kotlin
@Multipart
@POST("user/avatar")
suspend fun uploadAvatar(
    @Part avatar: MultipartBody.Part
): Result<AvatarResponse>
```

**完整 URL**：`POST http://10.237.36.80:8080/api/user/avatar`

---

### 2. 请求格式

#### Content-Type
```
multipart/form-data; boundary=----WebKitFormBoundary...
```

#### Form Data 字段
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| avatar | File | 头像图片文件 | `compressed_1234567890.jpg` |

#### 文件要求
- **格式**：JPEG
- **尺寸**：最大 200x200 像素（前端已自动压缩）
- **质量**：85%（前端已压缩）
- **大小**：通常 < 50KB（压缩后）
- **文件名**：`compressed_{timestamp}.jpg`

#### 认证方式
通过 OkHttp Interceptor 自动添加 Header：
```
Authorization: Bearer {token}
```

---

### 3. 前端发送代码

**文件位置**：`app/src/main/java/com/example/myapplication/presentation/profile/ProfileViewModel.kt`

```kotlin
// 1. 压缩图片（200x200, 85% 质量）
val compressedFile = compressImage(file)

// 2. 构建 Multipart 请求体
val requestBody = compressedFile.asRequestBody("image/jpeg".toMediaTypeOrNull())
val filePart = MultipartBody.Part.createFormData(
    "avatar",              // ⭐ 字段名必须是 "avatar"
    compressedFile.name,   // 文件名
    requestBody            // 文件内容
)

// 3. 发送请求
val response = api.uploadAvatar(filePart)
```

---

### 4. 期望的响应格式

#### 成功响应（HTTP 200）
```json
{
  "code": 200,
  "message": "上传成功",
  "data": "/api/user/avatar/avatar_123456.jpg"
}
```

**或者**（绝对路径）：
```json
{
  "code": 200,
  "message": "上传成功",
  "data": "http://10.237.36.80:8080/api/user/avatar/avatar_123456.jpg"
}
```

#### 失败响应（HTTP 400/500）
```json
{
  "code": 400,
  "message": "文件格式不支持",
  "data": null
}
```

---

### 5. 前端解析逻辑

```kotlin
if (response.isSuccess()) {
    // ⭐ 从 AvatarResponse 中提取 URL 字符串
    val avatarUrl = response.data?.getDataAsString()
    
    if (!avatarUrl.isNullOrBlank()) {
        // 处理相对路径和绝对路径
        val fullUrl = when {
            avatarUrl.startsWith("http://") || avatarUrl.startsWith("https://") -> 
                avatarUrl
            avatarUrl.startsWith("/api/") -> 
                "http://10.237.36.80:8080$avatarUrl"
            avatarUrl.startsWith("/") -> 
                "http://10.237.36.80:8080/api$avatarUrl"
            else -> 
                "http://10.237.36.80:8080/api/$avatarUrl"
        }
        
        // 更新 UI
        _successMessage.value = "头像上传成功"
        loadProfile()  // 重新加载用户信息
    }
}
```

## ⚠️ 注意事项（后端必读）

### 1. 字段名必须匹配
- ✅ 正确：`@Part avatar: MultipartBody.Part` → 字段名为 `"avatar"`
- ❌ 错误：字段名为 `"file"`、`"image"`、`"photo"` 等

### 2. Content-Type 检查
确保后端能正确处理 `multipart/form-data` 请求，而不是期待 `application/json`。

### 3. 文件大小限制
- 前端已压缩到 < 50KB
- 建议后端限制：最大 5MB（防止恶意上传）

### 4. 文件存储路径
前端期望返回的 URL 能通过以下接口访问：
```
GET /api/user/avatar/{filename}
```

例如：
- 返回：`/api/user/avatar/avatar_123.jpg`
- 访问：`http://10.237.36.80:8080/api/user/avatar/avatar_123.jpg`

### 5. 认证 Token
确保后端从 `Authorization: Bearer {token}` 中正确提取用户 ID，并将头像关联到对应用户。

### 6. 数据库更新
上传成功后，务必更新用户表中的 `avatar` 字段，否则下次查询用户信息时仍返回旧头像。

在后端控制器入口处添加日志：
```java

    log.info("文件名: {}", file.getOriginalFilename());
    log.info("Content-Type: {}", file.getContentType());
    // ...
}
```

### Step 2: 检查参数解析
确认 Spring Boot 能正确解析 `MultipartFile`：
```java
// 确保使用 @RequestParam("avatar") 而不是其他名称
@RequestParam("avatar") MultipartFile file
```

### Step 3: 检查响应格式
确保返回 JSON 格式与前端期望一致：
```java
return Result.success("/api/user/avatar/" + filename);
```

### Step 4: 检查跨域配置（如果需要）
如果前后端分离部署，确保 CORS 配置允许 `multipart/form-data`。

## 🐛 当前问题

**现象**：以上日志完全没有输出，说明：
1. 可能请求未发送到后端
2. 或者后端返回了异常响应导致前端崩溃
3. 或者网络层拦截器出现问题

**需要后端协助排查**：
1. 后端是否收到请求？（检查访问日志）
2. 如果收到，返回了什么响应？
3. 如果没有收到，可能是网络问题或路由配置问题
