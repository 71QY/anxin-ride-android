# 《安心出行》适老化智能出行平台 - 完整项目设计思路(前后端分离版)

> **说明**:本文档已根据实际前端项目代码进行全面修正,保留了后端设计的完整性,仅修正了前端相关的不准确描述。

---

## 一、架构设计理念

本项目采用**纯后端智能体架构**,所有 AI 逻辑和业务处理在后端完成,**前端只负责展示和简单调用**。核心思想是降低前端复杂度,提升系统可维护性和扩展性。

架构分层清晰,从上到下分为四层:**表现层、业务层、数据层、外部服务层**。各层职责明确,通过依赖注入解耦。

---

## 二、核心模块划分

### 1. 智能体模块(Agent Module) - 后端

这是整个系统的核心大脑,负责理解用户意图并调度各种服务。包含意图识别、对话管理、图片处理、方言翻译等功能。通过通义千问大模型实现自然语言理解,配合本地规则引擎作为降级保障。

### 2. 订单模块(Order Module) - 后端

管理用户出行订单的完整生命周期,从创建、派单、进行到完成。支持普通订单和亲情守护的代叫车订单,集成司机模拟分配和位置推送功能。

### 3. 地图模块(Map Module) - 后端

封装腾讯地图和高德地图 API 调用,提供 POI 搜索、路线规划、逆地理编码等服务。采用腾讯优先策略,结果不足时自动降级到高德,确保搜索成功率。

### 4. 用户模块(User Module) - 后端

处理用户认证、信息管理、头像上传、紧急联系人等功能。集成 JWT 认证和 BCrypt 密码加密,保障用户数据安全。

### 5. 亲情守护模块(Guard Module) - 后端

专为长辈用户设计,支持亲友绑定、代叫车、三方群聊等功能。**通过后端返回的用户类型标志位,前端实现长辈精简模式的 UI 适配**。

## 四、核心业务流程设计

### 1. 用户对话交互流程

**后端流程**(保持不变):
用户输入自然语言后,系统先尝试用通义千问 AI 识别意图,分为搜索、确认、下单、聊天四种类型。AI 失败时自动降级到本地关键词匹配。识别成功后执行对应策略,搜索结果会缓存到 Redis,等待用户确认。用户确认后返回路线信息,再次明确下单才创建订单,实现两次确认防误操作。

**前端实现**(修正):
1. **语音输入**
   - 用户点击麦克风按钮
   - 启动百度语音 SDK(主用)或讯飞语音 SDK(备用)
   - 实时显示部分识别结果(`onPartialResult`)
   - 最终结果通过 `parseVoiceResult()` 解析 JSON
   
2. **文本发送**
   - 用户输入文字或语音转文字
   - 构建 WebSocket 消息:`{type:"chat", sessionId, content, lat, lng}`
   - 通过 `ChatWebSocketClient.sendRaw()` 发送
   
3. **接收响应**
   - 监听 WebSocket 消息流(`SharedFlow<String>`)
   - 使用 `kotlinx.serialization` 解析 JSON
   - 根据 `type` 字段分发处理:
     - `search`:显示 POI 列表卡片
     - `chat`:显示 AI 回复文本
     - `order`:显示订单创建成功提示
     - `error`:显示错误信息
   
4. **POI 选择**
   - 用户点击 POI 卡片
   - 前端发送确认消息:`{type:"confirm", content:poiName}`
   - 后端返回路线信息,前端渲染地图路线

### 2. 图片识别叫车流程

**后端流程**(保持不变):
用户拍摄图片上传 Base64 编码,后端调用通义千问 qwen-vl-plus 模型进行 OCR 识别。提取文字后判断是否为地址类(包含路、街、号等关键词),是则触发 POI 搜索,否则返回图片描述。支持最多 5 张图片批量识别,AI 会理解多张图片的关系。

**前端实现**(修正):
1. **图片选择与压缩**
   - 用户点击相机图标,调用系统相机或相册
   - 获取 Uri 后解码为 Bitmap
   - 压缩策略:
     - OCR 场景:最大 1920x1080,质量 85%
     - 聊天图片:动态压缩到 800KB 以内
   - 使用 `Bitmap.createScaledBitmap` 保持宽高比
   
2. **Base64 编码**
   - 移除 `data:image/jpeg;base64,` 前缀
   - 异步执行编码,避免阻塞 UI 线程
   - 日志输出压缩前后大小,便于调试
   
3. **多图批量发送**
   - 待发送列表(`pendingImages`),最多 3 张
   - 底部预览栏显示缩略图,支持删除
   - 一次性发送所有图片,附带文字说明
   - 构建消息:`{type:"image", imageBase64:[...], text:"..."}`
   
4. **OCR 结果展示**
   - 后端返回:`{success, type:"image_result", ocrText, places, message}`
   - 前端提取 OCR 文本并显示
   - 如果有 POI 列表,渲染为可点击卡片
   - 非地址类图片显示后端返回的自然语言描述
   
5. **内存管理**
   - 图片解码后立即释放原始 Bitmap
   - 使用 `remember` 缓存解码结果
   - 大图查看器支持滑动关闭,及时回收资源

### 3. 订单创建流程

**后端流程**(保持不变):
用户确认目的地后,系统获取起点坐标(优先使用前端传递,否则从缓存或 IP 定位获取),调用腾讯地图 API 规划驾车路线,获取真实距离和耗时。根据阶梯计价公式计算预估价格,创建订单记录并生成订单号。订单创建后通过 WebSocket 推送通知,模拟司机分配并启动位置推送。

**前端实现**(修正):
1. **目的地确认**
   - 用户从 POI 列表中选择目的地
   - 前端显示确认对话框(两次确认防误操作)
   - 用户点击"确认下单"
   
2. **订单创建请求**
   - **方式 A(推荐)**:通过 HTTP API
     ```kotlin
     apiService.createOrder(CreateOrderRequest(
         destName = poiName,
         destLat = lat,
         destLng = lng,
         userId = currentUserId
     ))
     ```
   - **方式 B(旧版)**:通过 WebSocket
     ```kotlin

         "type":"order",
         "content":"$poiName",

     ```
   
3. **订单状态追踪**
   - 订单创建成功后,跳转到行程追踪页面
   - 监听 WebSocket 推送的订单状态更新:
     - `DRIVER_REQUEST`:司机接单请求

4. **地图实时渲染**

   - 显示预计到达时间

## 六、安全与性能设计

### 1. 认证安全 - 后端

采用 JWT Token 认证,Token 有效期 7 天。HTTP 请求通过 AuthInterceptor 拦截验证,WebSocket 连接时从 URL 参数提取 Token 验证。密码使用 BCrypt 加密存储,防止明文泄露。

### 2. 前端 Token 管理

**实现方式**:
```kotlin
class TokenManager(private val context: Context) {
    private val dataStore = context.createDataStore(name = "auth_prefs")
    
    suspend fun saveToken(token: String) {
        dataStore.edit { prefs ->
            prefs[stringPreferencesKey("jwt_token")] = token
        }
    }
    
    fun getToken(): String? {
        return runBlocking {
            dataStore.data.first()[stringPreferencesKey("jwt_token")]
        }
    }
}
```

**自动附加 Token**:
```kotlin
class AuthInterceptor : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val token = MyApplication.tokenManager.getToken()
        val request = chain.request().newBuilder()
            .addHeader("Authorization", "Bearer $token")
            .build()
        return chain.proceed(request)
    }
}
```

### 3. 长辈模式保护 - 前端

**实现方式**:
- 通过 `isElderMode` 状态标志位控制 UI 显示
- 长辈端界面独立文件,修改不影响普通端
- 长辈端禁止某些复杂操作(如手动输入目的地)

### 4. 图片压缩优化 - 前端

**策略**:
- OCR 场景:最大 1920x1080,质量 85%(平衡清晰度和速度)
- 头像上传:最大 500x500,质量 85%(符合后端建议)
- 聊天图片:动态压缩到 800KB 以内(快速传输)

**实现**:
```kotlin
private fun compressImage(bitmap: Bitmap, maxSizeKB: Int): String {
    var quality = 85
    var compressed = bitmap
    var size = getBitmapSize(compressed)
    
    while (size > maxSizeKB && quality > 10) {
        quality -= 10
        val outputStream = ByteArrayOutputStream()
        compressed.compress(Bitmap.CompressFormat.JPEG, quality, outputStream)
        size = outputStream.size() / 1024
    }
    
    return Base64.encodeToString(outputStream.toByteArray(), Base64.DEFAULT)
}
```

### 5. WebSocket 智能重连 - 前端

**策略**:
- 指数退避:1s → 2s → 4s → 8s → 16s
- 最多重试 5 次
- 正常关闭不重连,异常关闭才重连
- 连接中状态标记,防止重复连接

**实现**:
```kotlin
private fun scheduleReconnect(sid: String, tk: String) {
    if (reconnectAttempts >= maxReconnectAttempts) {
        Log.e("WebSocket", "达到最大重连次数")
        return
    }
    
    val delay = backoffTimes.getOrNull(reconnectAttempts) ?: 16000L
    scope.launch {
        delay(delay)
        reconnectAttempts++
        Log.d("WebSocket", "🔄 第${reconnectAttempts}次重连")
        connect(sid, tk)
    }
}
```

### 典型请求流程

1. **用户发起请求**(前端)
   - 用户在 UI 输入文字或语音
   - ViewModel 构建 WebSocket 消息
   - `ChatWebSocketClient.sendRaw()` 发送

2. **后端接收处理**(后端)
   - WebSocket Handler 接收消息
   - Agent Module 识别意图
   - 调用 Map Module 搜索 POI
   - 结果存入 Redis

3. **后端返回响应**(后端)
   - 构建标准化响应 JSON
   - 通过 WebSocket 推送给前端

4. **前端展示结果**(前端)
   - `ChatViewModel` 监听消息流
   - 解析 JSON,提取数据
   - 更新 StateFlow 状态
   - Compose UI 自动刷新

5. **数据持久化**(后端)
   - 订单数据存入 MySQL
   - 会话状态存入 Redis
   - 形成冷热数据分离

## 八、异常处理与容错

### 1. 前端异常处理

**网络异常**:
```kotlin
try {
    val response = apiService.createOrder(request)
    if (response.isSuccessful) {
        // 处理成功
    } else {
        showError("请求失败:${response.code()}")
    }
} catch (e: Exception) {
    Log.e("HomeViewModel", "网络异常", e)
    showError("网络连接失败,请检查网络")
}
```

**WebSocket 断开**:
```kotlin
override fun onClosed(webSocket: WebSocket, code: Int, reason: String) {
    super.onClosed(webSocket, code, reason)
    isConnected = false
    
    if (code != 1000) {  // 非正常关闭
        Log.w("WebSocket", "连接异常断开,准备重连")
        scheduleReconnect(sid, tk)
    }
}
```

**语音识别失败**:
```kotlin
override fun onError(error: SpeechError?) {
    Log.e("BaiduSpeech", "识别错误:${error?.errorCode}")
    isListening = false
    
    // 尝试切换到讯飞语音
    if (useBaidu) {
        switchToIflytek()
    } else {
        onResult("")
    }
}
```

### 2. 后端容错保障(保持不变)

**三层容错**:
- 第一层:AI 意图识别(通义千问)
- 第二层:本地关键词匹配(Fallback)
- 第三层:默认聊天回复

任何一层失败都不会导致服务中断,确保系统高可用。同时加入防污染校验,检测 AI 返回的关键词是否合理,不合理则自动清洗。

**全局异常处理**:
全局异常处理器 GlobalExceptionHandler 捕获所有未处理异常,返回统一错误信息,防止敏感信息泄露。关键业务逻辑加入详细日志,便于问题排查。

## 九、性能优化策略

### 1. 前端性能优化

**Compose 性能**:
- 使用 `remember` 缓存计算结果
- `LazyColumn` 替代传统 ListView
- `derivedStateOf` 派生状态,减少重组
- `collectAsStateWithLifecycle` 生命周期感知

**图片优化**:
- 异步压缩和编码,避免阻塞 UI
- 使用 `remember` 缓存解码结果
- 大图查看器支持滑动关闭,及时回收

**网络优化**:
- OkHttp 连接池复用
- WebSocket 长连接,减少握手开销
- 请求去重,避免重复调用

### 2. 后端性能优化(保持不变)

**并行计算**:
POI 列表的路线计算使用 CompletableFuture 和自定义线程池并行执行,多个地点同时调用地图 API,将原本串行的耗时从 10 秒降至 2-3 秒。线程池配置核心 10 线程、最大 20 线程,队列容量 100,拒绝策略为 CallerRunsPolicy 保证任务不丢失。

**缓存策略**:
- POI 搜索结果缓存 10 分钟
- 会话状态缓存 30 分钟
- 用户位置缓存,避免频繁调用定位服务

**智能降级**:
腾讯地图优先(减少高德调用次数),结果充足时跳过高德搜索,既保证响应速度,又提高搜索覆盖率。

## 十、总结

### 前后端协作模式

| 功能模块 | 前端职责 | 后端职责 |
|---------|---------|---------|
| 语音识别 | SDK 集成、方言支持、实时显示 | - |
| 意图识别 | - | 通义千问 AI、关键词匹配 |
| 地图显示 | SDK 集成、相机控制、标记渲染 | - |
| POI 搜索 | 渲染列表、用户选择 | API 调用、排序算法、缓存 |
| 图片识别 | 压缩、编码、预览 | OCR 识别、地址判断 |
| 订单管理 | UI 展示、状态追踪 | 创建、派单、司机调度 |
| 用户认证 | Token 管理、自动附加 | JWT 验证、密码加密 |
| 长辈模式 | UI 适配、模式切换 | 用户类型标识、权限控制 |
| WebSocket | 连接管理、消息解析 | 会话管理、消息推送 |

### 核心优势

1. **前端轻量**:不涉及复杂业务逻辑,易于维护和测试
2. **后端强大**:集中处理 AI、地图、订单等核心功能
3. **职责清晰**:前后端通过标准化协议交互,解耦彻底
4. **可扩展性**:新增功能只需后端实现,前端仅需调整 UI
5. **高性能**:后端并行计算 + 缓存,前端响应式 UI

### 技术亮点

- ✅ **双语音引擎**:百度 + 讯飞,方言支持,实时显示
- ✅ **单例 WebSocket**:智能重连、心跳保活、跨页面共享
- ✅ **高德地图集成**:前台定位 + 后台追踪,相机优化
- ✅ **Jetpack Compose**:声明式 UI、响应式状态、性能优化
- ✅ **Hilt 依赖注入**:模块化设计、kapt 优化、VerifyError 修复
- ✅ **长辈模式**:双界面隔离、大字体高对比、快捷操作
- ✅ **图片优化**:多级压缩、批量发送、内存管理
