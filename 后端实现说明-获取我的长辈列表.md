# 获取我的长辈列表接口 - 后端实现说明

## 📋 接口概述

**接口名称**: 获取我的长辈列表  
**接口路径**: `GET /api/guard/myElders`  
**功能描述**: 普通用户(亲友)查询自己绑定的所有长辈列表  
**使用场景**: 普通用户在聊天页面显示"我的长辈"列表,点击后进入私聊

## 🔧 后端需要实现的内容

### 1. Controller层 - GuardianController

**文件位置**: `src/main/java/com/example/myapplication/controller/GuardianController.java`

```java
@RestController
@RequestMapping("/api/guard")
@RequiredArgsConstructor
public class GuardianController {
    
    private final GuardianService guardianService;
    
    /**
     * ⭐ 新增：获取我的长辈列表（普通用户操作）
     * GET /api/guard/myElders?userId={userId}
     * 
     * @param userId 当前登录用户ID（亲友ID）
     * @return 长辈列表
     */
    @GetMapping("/myElders")
    public Result<List<ElderInfo>> getMyElders(@RequestParam Long userId) {
        try {
            // 参数校验
            if (userId == null || userId <= 0) {
                return Result.error("用户ID不能为空");
            }
            
            List<ElderInfo> elderList = guardianService.getMyElders(userId);
            return Result.success(elderList);
            
        } catch (Exception e) {
            log.error("获取我的长辈列表失败", e);
            return Result.error("获取长辈列表失败：" + e.getMessage());
        }
    }
}
```

### 2. Service层 - GuardianService

**文件位置**: `src/main/java/com/example/myapplication/service/GuardianService.java`

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class GuardianService {
    
    private final UserRepository userRepository;
    private final GuardianRelationRepository guardianRelationRepository;
    
    /**
     * ⭐ 新增：获取我的长辈列表
     * 
     * @param guardianUserId 亲友用户ID
     * @return 长辈信息列表
     */
    public List<ElderInfo> getMyElders(Long guardianUserId) {
        log.info("🔍 开始获取亲友[{}]的长辈列表", guardianUserId);
        
        // 1. 校验用户是否存在
        User guardian = userRepository.findById(guardianUserId)
            .orElseThrow(() -> new BusinessException("用户不存在"));
        
        // 2. 查询该亲友绑定的所有长辈关系
        List<GuardianRelation> relations = guardianRelationRepository
            .findByGuardianUserId(guardianUserId);
        
        if (relations == null || relations.isEmpty()) {
            log.info("⚠️ 亲友[{}]暂未绑定任何长辈", guardianUserId);
            return Collections.emptyList();
        }
        
        log.info("✅ 找到{}个绑定关系", relations.size());
        
        // 3. 转换为 ElderInfo 列表
        List<ElderInfo> elderList = relations.stream()
            .map(relation -> {
                // 查询长辈用户信息
                User elderUser = userRepository.findById(relation.getElderUserId())
                    .orElse(null);
                
                if (elderUser == null) {
                    log.warn("⚠️ 长辈用户[{}]不存在，跳过", relation.getElderUserId());
                    return null;
                }
                
                // 构建 ElderInfo
                ElderInfo elderInfo = new ElderInfo();
                elderInfo.setUserId(elderUser.getId());
                elderInfo.setGuardId(relation.getId());  // ⭐ 绑定关系ID（用于解绑）
                elderInfo.setPhone(elderUser.getPhone());
                elderInfo.setName(elderUser.getNickname() != null ? elderUser.getNickname() : elderUser.getName());
                elderInfo.setStatus(relation.getStatus());  // 0-待激活，1-已绑定
                elderInfo.setBindTime(relation.getCreateTime() != null ? 
                    relation.getCreateTime().toString() : null);
                
                return elderInfo;
            })
            .filter(Objects::nonNull)  // 过滤掉null值
            .collect(Collectors.toList());
        
        log.info("✅ 成功返回{}个长辈信息", elderList.size());
        return elderList;
    }
}
```

     * 检查是否已经绑定过

     * @param elderUserId 长辈用户ID

### 4. 实体类 - GuardianRelation

**文件位置**: `src/main/java/com/example/myapplication/entity/GuardianRelation.java`

```java
@Entity
@Table(name = "guardian_relation")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class GuardianRelation {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;  // 绑定关系ID
    
    @Column(name = "guardian_user_id", nullable = false)
    private Long guardianUserId;  // 亲友用户ID
    
    @Column(name = "elder_user_id", nullable = false)
    private Long elderUserId;  // 长辈用户ID
    
    @Column(name = "relationship")
    private String relationship;  // 与长辈关系（如：子女、亲属、护工）
    
    @Column(name = "status", nullable = false)
    private Integer status;  // 0-待激活，1-已绑定
    
    @Column(name = "create_time")
    private LocalDateTime createTime;  // 绑定时间
    
    @Column(name = "update_time")
    private LocalDateTime updateTime;  // 更新时间
    
    @PrePersist
    protected void onCreate() {
        createTime = LocalDateTime.now();
        updateTime = LocalDateTime.now();
        if (status == null) {
            status = 1;  // 默认已绑定
        }
    }
    
    @PreUpdate
    protected void onUpdate() {
        updateTime = LocalDateTime.now();
    }
}
```

### 6. 数据库表结构

**表名**: `guardian_relation`

```sql
CREATE TABLE IF NOT EXISTS guardian_relation (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '绑定关系ID',
    guardian_user_id BIGINT NOT NULL COMMENT '亲友用户ID',
    elder_user_id BIGINT NOT NULL COMMENT '长辈用户ID',
    relationship VARCHAR(50) COMMENT '与长辈关系（如：子女、亲属、护工）',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-待激活，1-已绑定',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '绑定时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    -- 索引
    INDEX idx_guardian_user_id (guardian_user_id),
    INDEX idx_elder_user_id (elder_user_id),
    
    -- 唯一约束：同一亲友不能重复绑定同一长辈
    UNIQUE KEY uk_guardian_elder (guardian_user_id, elder_user_id),
    
    -- 外键约束
    FOREIGN KEY (guardian_user_id) REFERENCES user(id) ON DELETE CASCADE,
    FOREIGN KEY (elder_user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='亲情守护绑定关系表';
```

## 📊 业务逻辑说明

### 1. 绑定流程

当亲友通过"帮长辈注册"或"绑定已有长辈"功能时:

```java
// 在 GuardianService.registerElder() 或 bindExistingElder() 中
private void createGuardianRelation(Long guardianUserId, Long elderUserId, String relationship) {
    // 1. 检查数量限制
    long elderCount = guardianRelationRepository.countByGuardianUserId(guardianUserId);
    if (elderCount >= 99) {
        throw new BusinessException("您已绑定99个长辈，达到上限");
    }
    
    long guardianCount = guardianRelationRepository.countByElderUserId(elderUserId);
    if (guardianCount >= 4) {
        throw new BusinessException("该长辈已有4位亲友守护，达到上限");
    }
    
    // 2. 检查是否已绑定
    if (guardianRelationRepository.existsByGuardianUserIdAndElderUserId(guardianUserId, elderUserId)) {
        throw new BusinessException("您已经绑定过该长辈账号");
    }
    
    // 3. 创建绑定关系
    GuardianRelation relation = new GuardianRelation();
    relation.setGuardianUserId(guardianUserId);
    relation.setElderUserId(elderUserId);
    relation.setRelationship(relationship);
    relation.setStatus(1);  // 已绑定
    
    guardianRelationRepository.save(relation);
}
```

---

### 2. 解绑流程

当亲友解绑某个长辈时:

```java
@Transactional
public void unbindElder(Long guardianUserId, Long guardId) {
    // 1. 查找绑定关系
    GuardianRelation relation = guardianRelationRepository.findById(guardId)
        .orElseThrow(() -> new BusinessException("绑定关系不存在"));
    
    // 2. 验证是否是本人的绑定关系
    if (!relation.getGuardianUserId().equals(guardianUserId)) {
        throw new BusinessException("无权解绑该长辈");
    }
    
    // 3. 删除绑定关系
    guardianRelationRepository.delete(relation);
    
    log.info("✅ 亲友[{}]已解绑长辈[{}]", guardianUserId, relation.getElderUserId());
}
```

## ⚠️ 注意事项

1. **权限控制**: 确保只能查询自己的长辈列表,不能查询他人的
2. **数据一致性**: 删除用户时,级联删除相关的绑定关系
3. **性能优化**: 如果长辈列表很大,考虑添加分页
4. **缓存策略**: 可以考虑缓存长辈列表,减少数据库查询
5. **日志记录**: 记录查询操作,便于排查问题
