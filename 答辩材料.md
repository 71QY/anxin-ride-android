## 项目概述

### 项目名称
**安心出行 (Anxin Travel)** - Android 智能出行服务平台

### 项目定位
基于 Jetpack Compose 的现代化 Android 应用，提供智能语音交互、地图导航、订单管理等一站式出行服务。

### 核心特性
- 🎯 **智能语音助手**: 集成讯飞语音识别 + WebSocket 实时通信
- 🗺️ **高德地图集成**: 精准定位、POI 搜索、路线规划
- 💬 **AI 对话系统**: 基于通义千问的智能体交互
- 📱 **Material Design 3**: 现代化 UI/UX 设计
- 🔐 **安全认证**: Token 管理、用户隐私保护

### 开发环境
- **语言**: Kotlin 1.9.22
- **最低 SDK**: API 26 (Android 8.0)
- **目标 SDK**: API 34 (Android 14)
- **构建工具**: Gradle 8.1.0 + AGP 8.1.0
- **JVM 版本**: Java 17

### MVVM 架构说明

#### 1. **Presentation Layer (表现层)**
- **Screen**: Jetpack Compose UI 组件
- **ViewModel**: 状态管理和业务逻辑协调
- **StateFlow**: 单向数据流，响应式更新

#### 2. **Data Layer (数据层)**
- **Repository**: 统一数据访问接口
- **ApiService**: Retrofit 网络请求封装
- **WebSocketClient**: 实时通信管理

#### 3. **Core Layer (核心层)**
- **Hilt**: 依赖注入框架
- **DataStore**: 轻量级本地存储
- **Coroutines**: 协程异步处理

### 设计模式应用

| 设计模式 | 应用场景 | 优势 |
|---------|---------|------|
| **MVVM** | 整体架构 | 关注点分离，可测试性强 |
| **Repository** | 数据访问抽象 | 解耦数据源，便于替换 |
| **Singleton** | WebSocket 客户端 | 全局唯一连接 |

### 4. **WebSocket 实时通信**

#### 核心特性
- 🔄 **自动重连**: 指数退避算法（1s → 2s → 4s → 8s → 16s）
- ❤️ **心跳保活**: 30 秒间隔发送 ping
- 🛡️ **异常处理**: 详细的错误日志和诊断
- 📊 **消息队列**: Channel 缓冲，避免消息丢失

#### 连接管理
```kotlin
@Singleton
class ChatWebSocketClient @Inject constructor() {
    private val client = OkHttpClient.Builder()
        .pingInterval(30, TimeUnit.SECONDS)
        .build()
    
    private var reconnectAttempts = 0
    private val maxReconnectAttempts = 5
    private val backoffTimes = listOf(1000L, 2000L, 4000L, 8000L, 16000L)
    
    fun connect(sessionId: String, token: String) {
        val url = "${BuildConfig.WEBSOCKET_URL}?token=$token&sessionId=$sessionId"
        
        val request = Request.Builder()
            .url(url)
            .addHeader("Sec-WebSocket-Protocol", "chat")
            .addHeader("X-Session-ID", sessionId)
            .build()
        
        webSocket = client.newWebSocket(request, object : WebSocketListener() {
            override fun onFailure(webSocket: WebSocket, t: Throwable, response: Response?) {
                isConnected = false
                
                if (reconnectAttempts < maxReconnectAttempts) {
                    val delayTime = backoffTimes.getOrElse(reconnectAttempts) { 16000L }
                    reconnectAttempts++
                    
                    scope.launch {
                        delay(delayTime)
                        connect(sessionId, token)
                    }
                }
            }
        })
    }
}
```

object RetrofitClient {
    private const val BASE_URL = BuildConfig.API_BASE_URL
    
    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor { chain ->
            val token = MyApplication.tokenManager.getTokenSync()

### 3. 高德地图 Compose 封装

```kotlin
@Composable
fun MapViewComposable(
    modifier: Modifier = Modifier,
    onMapReady: (AMap) -> Unit,
    onMapClick: (LatLng) -> Unit,
    onPoiClick: (Poi) -> Unit
) {
    val context = LocalContext.current
    val lifecycleOwner = LocalLifecycleOwner.current
    val mapView = remember { MapView(context) }
    var aMap by remember { mutableStateOf<AMap?>(null) }

    DisposableEffect(lifecycleOwner) {
        val observer = LifecycleEventObserver { _, event ->
            when (event) {
                Lifecycle.Event.ON_CREATE -> mapView.onCreate(Bundle())
                Lifecycle.Event.ON_RESUME -> mapView.onResume()
                Lifecycle.Event.ON_PAUSE -> mapView.onPause()
                Lifecycle.Event.ON_DESTROY -> mapView.onDestroy()
            }
        }
        lifecycleOwner.lifecycle.addObserver(observer)
        
        mapView.getMapAsync { map ->
            aMap = map
            map.setOnMapClickListener { latLng ->
                onMapClick(latLng)
            }
            map.setOnPoiClickListener { poi ->
                onPoiClick(poi)
            }
            onMapReady(map)
        }
        
        onDispose {
            lifecycleOwner.lifecycle.removeObserver(observer)
            mapView.onDestroy()
        }
    }

    AndroidView(
        factory = { mapView },
        modifier = modifier
    )
}
```

private fun compressBitmap(bitmap: Bitmap, maxSize: Int = 500 * 1024): ByteArray {
    var quality = 80
    var outputStream = ByteArrayOutputStream()
    
    bitmap.compress(Bitmap.CompressFormat.JPEG, quality, outputStream)
    
    // 如果超过最大大小，降低质量
    while (outputStream.size() > maxSize && quality > 10) {
        outputStream.reset()
        quality -= 10
        bitmap.compress(Bitmap.CompressFormat.JPEG, quality, outputStream)
    }
    
    return outputStream.toByteArray()
}

## Q&A 准备

### 常见问题预测

#### 1. 为什么选择 Jetpack Compose？
**答**: 
- 声明式 UI，代码更简洁（减少 40%+ 代码量）
- 状态驱动，自动更新，减少 bug
- Google 官方推荐，生态完善
- 更好的类型安全和编译时检查

#### 2. WebSocket 如何保证连接稳定性？
**答**:
- 指数退避重连算法（1s → 2s → 4s → 8s → 16s）
- 30 秒心跳保活机制
- 详细的错误日志和诊断
- Channel 缓冲消息，避免丢失

#### 3. 如何处理内存泄漏？
**答**:
- ViewModel 使用 `viewModelScope`，自动取消协程
- Compose 使用 `DisposableEffect` 管理生命周期
- 及时清理资源（`onCleared()` 方法）
- 避免在协程中持有 Activity/Fragment 引用

#### 4. 性能优化的关键点？
**答**:
- Compose 重组优化（remember、状态聚合）
- 网络请求防抖（Coroutine Job 取消）
- LRU 缓存地理编码结果
- 图片压缩上传
- Gradle 构建优化（并行、缓存）

#### 5. 项目中最大的技术挑战？
**答**:
- **WebSocket 稳定性**: 通过指数退避 + 心跳解决
- **地图生命周期**: 通过 LifecycleObserver 管理
- **状态管理复杂性**: 通过 StateFlow + 单向数据流简化
- **第三方 SDK 集成**: 仔细阅读文档，做好异常处理
