# 🚀 后端开发快速上手指南 - 长辈端收藏功能增强

## 📌 一句话总结

长辈点击收藏地点的"发送目的地"或"已到达"按钮后，后端需要通过WebSocket实时通知亲友。

---

## ⚡ 5分钟快速理解

### 场景1: 长辈发送目的地给亲友

```
长辈操作流程:
1. 打开收藏页面
2. 点击某个收藏地点的"📤 发送目的地"按钮
3. 选择要发送给哪位亲友（如：儿子李四）
4. 点击"发送"

后端需要做的:
1. 接收 POST /api/favorites/{favoriteId}/share-to-guardian 请求
2. 验证权限（必须是收藏地点拥有者）
3. 验证亲友关系（必须已绑定）
4. 通过WebSocket向亲友推送消息（类型：FAVORITE_DESTINATION_SHARED）
5. 返回成功响应

亲友收到WebSocket消息后:
- 显示代叫车确认对话框
- 可以点击"帮TA叫车"直接创建订单
```

### 场景2: 长辈确认到达目的地

```
长辈操作流程:
1. 到达收藏地点（如：医院）
2. 点击"✅ 已到达"按钮

后端需要做的:
1. 接收 POST /api/favorites/{favoriteId}/confirm-arrival 请求
2. 更新 user_favorites.last_visited_at = NOW()
3. 在 favorite_visit_history 表插入访问记录
4. 通过WebSocket向所有绑定的亲友推送消息（类型：ARRIVAL_CONFIRMED）
5. 返回成功响应

亲友收到WebSocket消息后:
- 显示通知："张三已到达市人民医院"
- 不用反复打电话确认
```

### ✅ 第2步：新增API接口（预计2小时）

#### 接口1: 分享收藏地点给亲友

```java
@PostMapping("/{favoriteId}/share-to-guardian")
public ApiResponse<Map<String, Object>> shareToGuardian(
        @PathVariable Long favoriteId,
        @RequestBody ShareFavoriteRequest request,
        HttpServletRequest httpRequest) {
    
    Long userId = getUserIdFromToken(httpRequest);
    
    // 1. 验证收藏地点存在且属于该用户
    FavoriteLocation favorite = favoriteRepository.findByIdAndUserId(favoriteId, userId)
        .orElseThrow(() -> new BusinessException("收藏地点不存在"));
    
    // 2. 验证亲友关系
    GuardianRelationship relationship = guardianRepository.findByElderIdAndGuardianId(userId, request.getGuardianUserId())
        .orElseThrow(() -> new BusinessException("未找到与该亲友的绑定关系"));
    
    // 3. 构建WebSocket消息
    Map<String, Object> wsMessage = new HashMap<>();
    wsMessage.put("type", "FAVORITE_DESTINATION_SHARED");
    wsMessage.put("elderId", userId);
    wsMessage.put("elderName", getUserName(userId));
    wsMessage.put("favoriteId", favoriteId);
    wsMessage.put("destinationName", favorite.getName());
    wsMessage.put("destinationAddress", favorite.getAddress());
    wsMessage.put("latitude", favorite.getLatitude());
    wsMessage.put("longitude", favorite.getLongitude());
    wsMessage.put("phone", favorite.getPhone());
    wsMessage.put("description", favorite.getDescription());
    wsMessage.put("timestamp", System.currentTimeMillis());
    
    // 4. 推送给亲友
    webSocketService.sendMessageToUser(request.getGuardianUserId(), wsMessage);
    
    // 5. 返回响应
    Map<String, Object> data = new HashMap<>();
    data.put("favoriteId", favoriteId);
    data.put("guardianUserId", request.getGuardianUserId());
    data.put("sharedAt", new Date());
    
    return ApiResponse.success(data);
}
```

#### 接口2: 确认到达目的地

```java
@PostMapping("/{favoriteId}/confirm-arrival")
public ApiResponse<Map<String, Object>> confirmArrival(
        @PathVariable Long favoriteId,
        @RequestBody(required = false) ConfirmArrivalRequest request,
        HttpServletRequest httpRequest) {
    
    Long userId = getUserIdFromToken(httpRequest);
    Long orderId = request != null ? request.getOrderId() : null;
    
    // 1. 验证收藏地点存在且属于该用户
    FavoriteLocation favorite = favoriteRepository.findByIdAndUserId(favoriteId, userId)
        .orElseThrow(() -> new BusinessException("收藏地点不存在"));
    
    // 2. 更新最后访问时间
    favorite.setLastVisitedAt(new Date());
    favoriteRepository.save(favorite);
    
    // 3. 记录访问历史
    VisitHistory history = new VisitHistory();
    history.setUserId(userId);
    history.setFavoriteId(favoriteId);
    history.setVisitTime(new Date());
    history.setLatitude(favorite.getLatitude());
    history.setLongitude(favorite.getLongitude());
    history.setOrderId(orderId);
    visitHistoryRepository.save(history);
    
    // 4. 获取所有绑定的亲友
    List<GuardianRelationship> guardians = guardianRepository.findByElderId(userId);
    
    // 5. 构建WebSocket消息
    Map<String, Object> wsMessage = new HashMap<>();
    wsMessage.put("type", "ARRIVAL_CONFIRMED");
    wsMessage.put("elderId", userId);
    wsMessage.put("elderName", getUserName(userId));
    wsMessage.put("favoriteId", favoriteId);
    wsMessage.put("destinationName", favorite.getName());
    wsMessage.put("orderId", orderId);
    wsMessage.put("arrivedAt", new SimpleDateFormat("yyyy-MM-dd HH:mm:ss").format(new Date()));
    wsMessage.put("latitude", favorite.getLatitude());
    wsMessage.put("longitude", favorite.getLongitude());
    wsMessage.put("timestamp", System.currentTimeMillis());
    
    // 6. 推送给所有亲友
    List<Long> notifiedGuardians = new ArrayList<>();
    for (GuardianRelationship guardian : guardians) {
        webSocketService.sendMessageToUser(guardian.getGuardianId(), wsMessage);
        notifiedGuardians.add(guardian.getGuardianId());
    }
    
    // 7. 返回响应
    Map<String, Object> data = new HashMap<>();
    data.put("favoriteId", favoriteId);
    data.put("confirmedAt", new Date());
    data.put("notifiedGuardians", notifiedGuardians);
    
    return ApiResponse.success(data);
}
```

## ⚠️ 常见问题

### Q1: WebSocket推送失败怎么办？
**A**: 检查亲友是否在线，如果不在线可以：
- 方案1: 保存离线消息，等上线后推送
- 方案2: 发送短信通知（需要集成短信服务）

### Q2: 如何防止重复推送？
**A**: 在前端调用API时添加防抖，后端可以：
- 记录最近一次推送时间
- 相同内容在N秒内不重复推送

### Q3: 访问历史表数据量太大怎么办？
**A**: 
- 定期清理过期数据（如保留1年）
- 按月分表
- 归档到冷存储
