# 忘记密码功能 - 后端 API 接口注意事项

## 📋 功能流程说明

### 用户操作流程
1. **第一步**：用户在登录页点击"忘记密码"，进入重置密码流程
2. **第二步**：输入手机号 → 点击"获取验证码" → 输入验证码 → 点击"验证并重置"
3. **第三步**：验证成功后跳转到重置密码页面 → 输入新密码和确认密码 → 点击"重置密码"完成

---

## 🔌 涉及的 API 接口

### 1. 发送验证码接口

**接口路径**: `POST /api/auth/sendCode`

**请求参数**:
```json
{
  "phone": "13800138000"
}
```

**响应格式**:
```json
{
  "code": 200,
  "message": "验证码发送成功",
  "success": true,
  "timestamp": 1717516800000
}
```

**⚠️ 注意事项**:
- ✅ 验证码有效期建议设置为 **5-10 分钟**
- ✅ 需要实现防刷机制（同一手机号 60 秒内只能发送一次）
- ✅ 验证码应为 **6 位数字**
- ✅ 如果手机号未注册，也应返回成功提示（避免泄露用户信息）

---

### 2. 验证验证码并重置密码接口

**接口路径**: `POST /api/auth/forgotPassword`

**请求参数**:
```json
{
  "phone": "13800138000",
  "code": "123456",
  "newPassword": "newPass123!"
}
```

**响应格式**:
```json
{
  "code": 200,
  "message": "密码重置成功",
  "success": true,
  "data": {
    "userId": "12345",
    "phone": "138****8000"
  },
  "timestamp": 1717516800000
}
```

**⚠️ 注意事项**:
- ✅ 验证码校验失败时，应返回明确的错误信息（如"验证码错误或已过期"）
- ✅ 密码策略：**至少 8 位，包含字母和特殊符号**
- ✅ 验证成功后应使旧验证码立即失效
- ✅ 建议记录密码修改日志（用于安全审计）
- ✅ 密码需要加密存储（BCrypt 或类似算法）

**❌ 错误响应示例**:
```json
{
  "code": 400,
  "message": "验证码错误或已过期",
  "success": false,
  "timestamp": 1717516800000
}
```

## 🔒 安全要求

### 1. 验证码安全
- ⚠️ **必须验证验证码的有效期**
- ⚠️ **连续错误 5 次后应锁定该手机号**（防止暴力破解）
- ⚠️ 验证码使用后应立即失效
- ⚠️ 验证码在服务器端应加密存储

### 2. 密码安全
- ⚠️ **禁止明文传输密码**（前端应使用 HTTPS）
- ⚠️ **禁止明文存储密码**（后端应使用 BCrypt 等加盐哈希算法）
- ⚠️ 密码策略校验应在后端再次执行

### 3. 防刷机制
- ⚠️ 同一手机号 **60 秒内只能发送一次验证码**
- ⚠️ 同一 IP 地址 **1 小时内最多发送 10 次验证码**
- ⚠️ 检测到异常行为时应触发风控（如短信轰炸）

## 🐛 常见问题与解决方案

### Q1: 验证码一直收不到？
**可能原因**:
- 短信服务商故障
- 手机号格式错误
- 运营商拦截

**解决方案**:
- 前端显示"验证码已发送"的模拟模式（开发环境）
- 提供语音验证码备选方案
- 显示客服联系方式

---

### Q2: 验证码校验总是失败？
**可能原因**:
- 验证码已过期
- 验证码已被使用
- 前后端验证码格式不一致

**解决方案**:
- 检查服务器时间与客户端时间是否同步
- 确保验证码比较时去除空格和特殊字符
- 统一使用字符串比较而非数字比较

---

### Q3: 密码重置成功但用户无法登录？
**可能原因**:
- 密码加密算法不一致
- Token 未及时刷新
- 缓存未清除

**解决方案**:
- 重置密码后强制用户重新登录
- 清除该用户的所有缓存 Token
- 检查密码加密算法是否一致

## 📝 测试用例建议

### 正常流程测试
1. ✅ 输入已注册的手机号 → 收到验证码 → 输入正确验证码 → 设置新密码 → 重置成功

2. ❌ 验证码错误 → 应提示"验证码错误或已过期"
3. ❌ 验证码过期（超过 10 分钟）→ 应提示"验证码已过期"
4. ❌ 密码长度不足 8 位 → 应提示"密码至少 8 位"
5. ❌ 密码只包含数字 → 应提示"密码必须包含字母和特殊符号"
6. ❌ 两次密码输入不一致 → 前端校验不通过
7. ❌ 60 秒内重复发送验证码 → 应提示"操作过于频繁"

## 🚀 性能优化建议

1. **验证码生成**: 使用线程安全的随机数生成器
2. **验证码存储**: 使用 Redis 等高性能缓存，设置 TTL
3. **短信发送**: 异步发送，避免阻塞主线程
4. **数据库操作**: 密码更新使用事务，确保数据一致性
