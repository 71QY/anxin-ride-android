# 《安心出行》适老化智能出行平台 - 项目简介(前端修正版)

安心出行是一款基于 AI 智能体的智慧出行服务平台,将阿里云通义千问大模型与地图服务深度融合,为用户提供自然语言交互的智能化打车体验。

## 核心功能

**AI 智能体对话交互**:用户通过自然语言(如"帮我找附近的医院")表达出行需求,AI 自动理解意图,完成地点搜索、路线规划、订单创建。支持多轮对话上下文,可追问"换一个"、"远不远"。**前端集成百度语音 SDK 和讯飞语音 SDK,支持普通话、粤语、四川话等多种方言实时识别**。

**图片识别智能叫车**:拍摄医院招牌、路牌、名片,AI 通过 OCR 自动提取地址并搜索。**前端支持最多 3 张图片同时上传,提供底部预览栏和删除功能,自动压缩图片到合适尺寸**。

**多源地图智能搜索**:集成腾讯地图(优先)和高德地图(补充)双数据源。支持关键词纠偏(如"老校区"转为"韩山师范学院东区"),内置 50+ 知名地标强制识别(故宫、北大、清华等)。**前端使用高德地图 SDK 11.1.0,负责地图渲染、定位服务和 POI 列表展示**。

**POI 智能推荐排序**:采用多维度评分算法:名称匹配占 50%、类型匹配占 30%、距离权重占 20%。内置同城加成机制,优先推荐本地地点。**前端只负责渲染后端返回的已排序结果**。

**亲情守护模式**:长辈精简 UI 提供大字体(30sp)、高对比度(蓝色 #1677FF)、简化操作(72dp 按钮)。支持亲友远程代叫车,AI 自动推送通知给长辈。**前端实现双界面隔离架构(ElderSimplifiedScreen vs NormalHomeScreen),通过 userType 字段自动切换模式**。

**语音交互服务**:**前端支持语音转文字输入叫车需求(百度/讯飞双引擎),以及文字转语音播报重要信息(Android 系统 TTS,仅在长辈模式下启用)**。

## 技术特色

**千问双模型协同**:qwen-turbo 用于意图识别和方言翻译,qwen-vl-plus 用于图片 OCR 识别。

**三层 Fallback 保障**:AI 识别失败自动降级为关键词匹配,再失败则返回默认回复,确保服务不中断。

**并行路线计算**:使用 CompletableFuture 线程池,多个 POI 同时计算路线,速度提升 3 倍。

**WebSocket 实时通信**:平均响应时间小于 2 秒,提供流畅的对话体验。**前端实现单例 WebSocket 客户端,支持指数退避重连(1s→2s→4s→8s→16s)和心跳保活机制**。

**两次确认机制**:搜索后需用户确认,再明确下单,防止误操作。

## 技术栈

- **AI 引擎**:阿里云通义千问(比赛合规工具)
- **后端**:Java 17 + Spring Boot 3.1.5
- **地图服务**:腾讯地图 + 高德地图
- **数据库**:MySQL 8.0 + Redis
- **前端**:Android Kotlin + Jetpack Compose + Hilt
  - **语音识别**:百度语音 SDK + 讯飞语音 SDK
  - **地图服务**:高德地图 SDK 11.1.0
  - **网络通信**:Retrofit + OkHttp WebSocket
  - **数据持久化**:DataStore Preferences
  - **图片处理**:Coil + Bitmap 压缩
  - **TTS 播报**:Android TextToSpeech(可选)
