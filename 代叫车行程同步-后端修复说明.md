# 代叫车行程同步 - 后端修复说明

## 需求描述

普通用户（代叫人/亲友）应该能够**实时同步查看**长辈端的打车行程追踪界面，包括：
- 司机位置实时更新
- 订单状态变化（待确认→司机接单→行程中→已完成）
- 司机信息（姓名、电话、车牌等）
- ETA预估到达时间

## 当前问题

❌ 后端只将WebSocket消息推送给**长辈端**（userId）
❌ 普通用户（代叫人/guardianUserId）收不到行程更新消息
❌ 普通用户无法实时查看长辈的行程状态

## 期望行为

### 正常流程

```
1. 普通用户为长辈叫车
   ↓
2. 长辈确认订单
   ↓
3. 后端推送司机申请 → 同时发送给：
   • 长辈端 (userId)
   • 代叫人端 (guardianUserId)  ⭐ 新增
   ↓
4. 长辈同意司机
   ↓
5. 后端推送司机位置更新 → 同时发送给：
   • 长辈端 (userId)
   • 代叫人端 (guardianUserId)  ⭐ 新增
   ↓
6. 行程结束
   ↓
7. 两端都能完整看到整个行程过程
```

## 后端需要修改的地方

### 1. WebSocket消息推送逻辑

**核心原则**：所有与订单相关的WebSocket消息，都需要同时推送给**两个用户**：
- `userId`（长辈/乘客）
- `guardianUserId`（代叫人/亲友）

#### 修改前（当前逻辑）

```java
// 伪代码 - 只推送给长辈
public void sendOrderUpdate(Long orderId, WsMessage message) {
    Order order = orderRepository.findById(orderId).orElse(null);
    if (order == null) return;
    
    // ❌ 只推送给长辈
    String elderSessionId = "user_" + order.getUserId();
    webSocketService.sendMessage(elderSessionId, message);
}
```

#### 修改后（需要实现）

```java
// 伪代码 - 同时推送给长辈和代叫人
public void sendOrderUpdate(Long orderId, WsMessage message) {
    Order order = orderRepository.findById(orderId).orElse(null);
    if (order == null) return;
    
    // ✅ 推送给长辈
    String elderSessionId = "user_" + order.getUserId();
    webSocketService.sendMessage(elderSessionId, message);
    
    // ✅ 新增：如果是代叫车订单，也推送给代叫人
    if (order.getGuardianUserId() != null && order.getGuardianUserId() > 0) {
        String guardianSessionId = "user_" + order.getGuardianUserId();
        webSocketService.sendMessage(guardianSessionId, message);
        
        log.info("✅ 已向代叫人 userId={} 推送订单 {} 的消息", 
            order.getGuardianUserId(), orderId);
    }
}
```

### 3. 统一推送工具方法

建议创建一个统一的推送工具类：

```java
@Component
public class OrderPushService {
    
    @Autowired
    private WebSocketService webSocketService;
    
    /**
     * 向订单相关的所有用户推送消息
     * @param orderId 订单ID
     * @param message WebSocket消息
     */
    public void pushToAllStakeholders(Long orderId, WsMessage message) {
        Order order = orderRepository.findById(orderId).orElse(null);
        if (order == null) {
            log.warn("⚠️ 订单 {} 不存在，无法推送", orderId);
            return;
        }
        
        // 1. 推送给乘客（长辈）
        String passengerSessionId = "user_" + order.getUserId();
        boolean passengerSent = webSocketService.sendMessage(passengerSessionId, message);
        
        if (passengerSent) {
            log.debug("✅ 已向乘客 userId={} 推送消息", order.getUserId());
        } else {
            log.warn("⚠️ 乘客 userId={} 未连接WebSocket", order.getUserId());
        }
        
        // 2. 如果是代叫车订单，推送给代叫人（亲友）
        if (order.getGuardianUserId() != null && order.getGuardianUserId() > 0) {
            String guardianSessionId = "user_" + order.getGuardianUserId();
            boolean guardianSent = webSocketService.sendMessage(guardianSessionId, message);
            
            if (guardianSent) {
                log.debug("✅ 已向代叫人 userId={} 推送消息", order.getGuardianUserId());
            } else {
                log.warn("⚠️ 代叫人 userId={} 未连接WebSocket", order.getGuardianUserId());
            }
        }
    }
    
    /**
     * 便捷方法：推送司机位置更新
     */
    public void pushDriverLocation(Long orderId, Driver driver, int etaMinutes) {
        WsMessage message = WsMessage.builder()
            .type("DRIVER_LOCATION")
            .orderId(orderId)
            .driverLat(driver.getCurrentLat())
            .driverLng(driver.getCurrentLng())
            .etaMinutes(etaMinutes)
            .build();
        
        pushToAllStakeholders(orderId, message);
    }
    
    /**
     * 便捷方法：推送司机接单请求
     */
    public void pushDriverRequest(Long orderId, Driver driver) {
        WsMessage message = WsMessage.builder()
            .type("DRIVER_REQUEST")
            .orderId(orderId)
            .driverId(driver.getId())
            .driverName(driver.getName())
            .driverPhone(driver.getPhone())
            .driverAvatar(driver.getAvatar())
            .carNo(driver.getCarNo())
            .carType(driver.getCarType())
            .carColor(driver.getCarColor())
            .rating(driver.getRating())
            .message("有新的司机接单，是否同意？")
            .build();
        
        pushToAllStakeholders(orderId, message);
    }
}
```

## 性能优化建议

### 1. 批量推送优化

如果有很多代叫车订单同时更新，可以考虑批量推送：

```java
// 不推荐：逐个推送
for (Order order : orders) {
    pushToBoth(order.getUserId(), order.getGuardianUserId(), message);
}

// 推荐：批量推送
Map<String, List<WsMessage>> batchMessages = new HashMap<>();
for (Order order : orders) {
    batchMessages.computeIfAbsent("user_" + order.getUserId(), k -> new ArrayList<>()).add(message);
    if (order.getGuardianUserId() != null) {
        batchMessages.computeIfAbsent("user_" + order.getGuardianUserId(), k -> new ArrayList<>()).add(message);
    }
}
batchMessages.forEach((sessionId, msgs) -> {
    webSocketService.sendBatchMessage(sessionId, msgs);
});
```

### 2. 频率控制

司机位置更新可能很频繁，可以适当降频：

```java
// 每5秒推送一次位置，而不是每秒
@Scheduled(fixedRate = 5000)
public void updateDriverLocations() {
    List<ActiveOrder> activeOrders = orderRepository.findActiveOrdersWithDrivers();
    for (ActiveOrder order : activeOrders) {
        Driver driver = driverRepository.findById(order.getDriverId()).orElse(null);
        if (driver != null) {
            int eta = calculateETA(driver, order);
            pushService.pushDriverLocation(order.getId(), driver, eta);
        }
    }
}
```
