# 后台定位追踪服务修复说明

## 问题诊断

### 核心问题
代叫车功能一直不好用的根本原因是：**应用切换到后台后，定位和WebSocket连接会断开**。

当你需要在长辈端和普通用户端之间切换测试时：
1. 切换到另一个应用后，Android系统可能会暂停或杀死当前应用
2. 定位服务停止，无法获取实时位置
3. WebSocket连接断开，无法接收代叫车请求通知
4. 导致代叫车功能失效

## 解决方案

### 1. 创建后台定位追踪服务

新建 `LocationTrackingService.kt`，实现以下功能：

- ✅ **前台服务**：显示持久化通知，防止被系统杀死
- ✅ **持续定位**：使用高德地图SDK，每5秒更新一次位置
- ✅ **WebSocket保持**：持续更新WebSocket客户端的位置信息
- ✅ **后台运行**：即使应用切换到后台，服务仍然运行

### 2. 修改AndroidManifest.xml

添加必要的权限和服务声明：

```xml
<!-- 后台定位服务权限 -->
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_LOCATION" />

<!-- 后台定位追踪服务 -->
<service
    android:name=".service.LocationTrackingService"
    android:enabled="true"
    android:exported="false"
    android:foregroundServiceType="location" />
```

### 3. 在HomeViewModel中添加控制方法

```kotlin
// 启动后台定位追踪服务
fun startBackgroundTracking()

// 停止后台定位追踪服务
fun stopBackgroundTracking()
```

### 4. 在UI层自动启动服务

#### 长辈端 (ElderSimplifiedScreen)
- 进入长辈模式并授予定位权限后，自动启动后台服务
- 确保长辈的位置能持续上报给后端

#### 普通用户端 (HomeScreen)
- 同样在进入页面时启动后台服务
- 确保持续接收代叫车请求通知

### 5. 在MainActivity.onDestroy中停止服务

避免资源浪费，在Activity销毁时停止后台服务。

## 技术细节

### LocationTrackingService 关键特性

1. **前台服务通知**
   - 通知ID: 1002
   - 渠道ID: location_tracking_channel
   - 低优先级，锁屏不可见
   - 点击通知可返回应用

2. **定位配置**
   - 模式：高精度（GPS + 网络）
   - 间隔：5秒
   - GPS优先
   - 不使用缓存
   - 超时时间：20秒

3. **位置更新处理**
   - 验证定位结果有效性
   - 更新WebSocket客户端位置
   - 记录详细日志便于调试

### 启动时机

```
用户进入长辈端/普通用户端
    ↓
检查定位权限
    ↓
启动独立定位 (startIndependentLocation)
    ↓
启动后台定位追踪服务 (startBackgroundTracking)
    ↓
服务在前台运行，持续获取位置
    ↓
即使切换到其他应用，服务仍继续运行
    ↓
退出应用时 onDestroy 停止服务
```

## 注意事项

### Android版本兼容性

- **Android 8.0+ (API 26+)**: 必须使用 `startForegroundService()`
- **Android 10+ (API 29+)**: 需要声明 `FOREGROUND_SERVICE_LOCATION` 权限
- **Android 12+ (API 31+)**: 需要更明确的前台服务类型声明

### 电池优化

- 前台服务会增加电量消耗
- 建议在不需要时及时停止服务
- 可以考虑添加智能启停逻辑（如：仅在有待处理订单时启动）

### 权限要求

用户需要授予以下权限：
1. `ACCESS_FINE_LOCATION` - 精确定位
2. `ACCESS_COARSE_LOCATION` - 粗略定位
3. 允许应用在后台运行（部分厂商需要手动设置）

### 厂商定制系统

某些国产手机（小米、华为、OPPO等）可能有额外的后台限制：
- 需要在系统设置中允许应用自启动
- 需要将应用加入白名单
- 可能需要关闭省电模式对应用的限制

## 后续优化建议

1. **智能启停**
   - 仅在有代叫车订单时启动服务
   - 订单完成后自动停止

2. **定位频率调整**
   - 移动时：5秒更新
   - 静止时：30秒更新
   - 根据速度动态调整

3. **电量优化**
   - 检测电池状态
   - 低电量时降低定位频率

4. **用户体验**
   - 自定义通知内容，显示当前位置
   - 提供快捷开关，允许用户手动启停

## 总结

通过这次修复，我们解决了代叫车功能的核心痛点：

**之前**：切换应用 → 定位停止 → WebSocket断开 → 功能失效  
**现在**：切换应用 → 后台服务继续运行 → 定位持续 → WebSocket保持 → 功能正常

现在你可以自由地在长辈端和普通用户端之间切换测试，无需担心功能失效！
